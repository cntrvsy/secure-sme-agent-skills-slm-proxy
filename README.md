# Securing SME Agent Skills via Fine-Tuned SLM Proxies

This repository contains the dissertation document, model fine-tuning pipeline, evaluation harness, and presentation slides for the research project: **"Securing SME Agent Skills via Fine-Tuned SLM Proxies"**.

---

## Project Overview

Small and Medium-sized Enterprises (SMEs) face an **"Operational Paradox"**: they must leverage AI automation to remain competitive, but they lack the computational and financial resources to deploy massive Large Language Models (LLMs) or manage complex Model Context Protocol (MCP) servers.

To bypass this overhead, SMEs adopt **Agent Skills**—lightweight, static, version-controlled directories containing a [SKILL.md](file:///mnt/1c5f09d2-4b36-42cf-8541-5cb64e85f9e9/projects/masters/dissartation%20doc/dissertation.typ#L39) file that defines instructions (YAML frontmatter + Markdown body) loaded into downstream agent context windows. However, these static skill files introduce severe vulnerabilities:

1. **Indirect Prompt Injection (IPI) Attacks:** Malicious payloads hidden within user reviews or external inputs that hijack the agent's behavior (e.g., executing exfiltration commands).
2. **The "Context Tax":** Inflation of the active prompt history due to verbose, unstructured skill documentation, increasing inference latency and API token costs.

### The Solution: Local SLM Security Proxy

We develop and evaluate a local security and semantic compression middleware artifact powered by a quantized 4-bit `Qwen2.5-3B-Instruct` Small Language Model (SLM) using the Unsloth framework. Running entirely locally on standard SME hardware (4–6 GB VRAM), the proxy intercepts custom skill directories, strips out-of-scope IPI payloads, and distills descriptive token overhead before transmitting content to downstream agents.

## SFT Training & Evaluation Pipeline

The machine learning implementation resides in the [dataset/](file:///mnt/1c5f09d2-4b36-42cf-8541-5cb64e85f9e9/projects/masters/dataset/) directory:

### 1. Dataset Generation

The corpus contains **3,276 virtual instances** generated by combinatoric cross-multiplication of benign SME workflows with malicious payloads (drawn from INJECAGENT and MSB benchmarks).

- Run [refine_dataset.py](file:///mnt/1c5f09d2-4b36-42cf-8541-5cb64e85f9e9/projects/masters/dataset/refine_dataset.py) to format the raw triples into system-user-response patterns mapping poisoned markdown structures onto clean "gold distilled" target skills.

### 2. Fine-Tuning Setup

LoRA/QLoRA fine-tuning is defined in [Qwen2_5_3B_SKILLmd_Sanitizer_Finetune.py](file:///mnt/1c5f09d2-4b36-42cf-8541-5cb64e85f9e9/projects/masters/dataset/Qwen2_5_3B_SKILLmd_Sanitizer_Finetune.py):

- **Base model:** quantized 4-bit `Qwen2.5-3B-Instruct`
- **PEFT framework:** Low-Rank Adaptation (LoRA) via `UnslothSFTTrainer`
- **Output:** Serialized GGUF and LoRA adapter weights for local execution.

### 3. Evaluation Harness

The test pipeline is managed by [evaluate_proxy.py](file:///mnt/1c5f09d2-4b36-42cf-8541-5cb64e85f9e9/projects/masters/dataset/evaluate_proxy.py):

- Simulates standard (Base) and jailbreak-enhanced (Enhanced) attacks across an unseen 1,198 holdout test split.
- Automatically calculates:
  - **ASR-valid:** Attack Success Rate (flags verbatim payload leaks or high keyword overlap).
  - **Format Integrity Pass:** Syntactic validity checks on YAML frontmatter & markdown blocks.
  - **Compression Ratio:** Token compression sizing using the `unsloth/Qwen2.5-3B-Instruct` tokenizer.
- Integrates local Ollama (`/api/generate`) and LM Studio (`/v1/chat/completions`) server endpoints.

## Interactive Svelte Slides (Animotion)

The presentation deck explaining this research is located under [src/](file:///mnt/1c5f09d2-4b36-42cf-8541-5cb64e85f9e9/projects/masters/src/) and built using **Animotion** (a Svelte-based animation layout framework).

### Running Slides Locally

1. **Install dependencies:**
   ```sh
   pnpm install
   ```
2. **Start the development server:**
   ```sh
   pnpm dev
   ```
   Open `http://localhost:5173/` in your browser.
3. **Build the production package:**
   ```sh
   pnpm build
   ```
   This generates a static build in the `.svelte-kit/output` folder.

### Key Presentation Features

- **Title Slide (100):** Custom-styled landing slide displaying the dissertation title and metadata.
- **Code Audit Visualizations (210, 500):** Code blocks showing vulnerable schemas and fine-tuning templates, styled in a highly readable light mockup container with solid black code text.
- **Dedicated Results Pages (610 to 640):** Separate slides displaying performance charts directly (Security Efficacy, Format Pass Rates, Token Compression, and the baseline's Truncation Illusion regression plots) with plenty of room to breathe.
